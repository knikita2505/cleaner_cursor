# Экранный модуль: Онбординг приложения

## Назначение
Онбординг — это ключевой эмоциональный мост между пользователем и приложением.  
Он должен:
- объяснить пользу приложения простыми и ясными словами,
- укрепить доверие к продукту,
- продемонстрировать «качество» и премиальность,
- мягко провести к разрешениям,
- подготовить к paywall,
- создать ощущение, что приложение — "эксперт по смартфону".

Стиль онбординга:
- спокойный, технологичный, без шума,
- приятные плавные градиенты,
- тонкая анимация, мягкие переходы,
- текст — короткий, умный, уверенный.

---

# Общая структура онбординга
Онбординг состоит из **4 основных экранов** (информационных) + 1 финального pre-permissions экрана.

Пользователь должен пройти через:
1. Экран ценности (главная миссия)
2. Экран преимуществ
3. Экран реальной выгоды (память, скорость, порядок)
4. Экран доверия и качества
5. Экран перед разрешениями (pre-permission warm-up)

После него → Permissions → Paywall A/B.

# 1. Экран: “Добро пожаловать в чистый и быстрый iPhone”

## Цель
Сформировать первое впечатление: приложение выглядит премиально, стабильно и безопасно.

### UI
- Большая плавная анимация круга (символ чистоты и оптимизации)
- Текст по центру:
  **“Давайте наведём порядок на вашем устройстве”**
  Подзаголовок:
  **“Умное сканирование фото, видео, дубликатов и системного захламления.”**

### Анимации:
- лёгкие волны света
- мягкое появление текста снизу

### Действия:
- Кнопка **Continue**

### Analytics:
- `onboarding_step_1_shown`
- `onboarding_step_1_next_tap`

# 2. Экран: “Освобождайте память — без потерь”

## Цель
Показать ключевое отличие: очистка без риска.

### UI
- Иллюстрация: архивирование больших видео или сравнение похожих фото
- Заголовок:
  **“Удаляем ненужное. Сохраняем важное.”**
- Текст:
  **“Точно находим дубликаты, похожие фото, Live и размытые кадры. Вы решаете, что оставить.”**

### Анимация:
- превью галереи с лёгким морфингом (меняется кадр)
- карточки “duplicate / low quality” появляются с fade-in

### Действия:
- Кнопка **Continue**

### Analytics:
- `onboarding_step_2_shown`
- `onboarding_step_2_next_tap`

# 5. Pre-permission экран: “Почти готово!”

## Цель
Подготовить пользователя к запросу разрешений.  
Это повышает CR разрешений на 20–40%.

### UI
- Большая иконка галереи
- Заголовок:
  **“Чтобы начать очистку, дайте доступ к вашим фото и видео”**
- Подзаголовок:
  **“Мы используем доступ только для анализа и удаления ненужных файлов. Всё остаётся на вашем устройстве.”**
- Маленькая строчка:
  **“Вы можете изменить это позже в настройках.”**

### Анимация:
- мягкий blur + появление иконки сверху

### CTA:
- **Enable Photo Access**
- Secondary: Skip for now

### При тапе:
- открывается системное permission окно (Photo Library)

### Analytics:
- `onboarding_pre_permission_shown`
- `onboarding_permission_request`
- отказ → `onboarding_permission_denied`
- разрешение → `onboarding_permission_granted`

---

# Финальный шаг после разрешений
После выдачи прав сразу показывается:
- Сканирование → 2–3 секунды прогресс-анимации
- Затем Paywall A/B

Это повышает CR в триал на 10–18%.

---

# Edge Cases
Если пользователь отказал в доступе:
- показываем экран:
  **“Без доступа к фото приложение не сможет находить дубликаты.”**
- CTA: Open Settings

---

# Переходы
- Экран 1 → Экран 2 → Экран 3 → Экран 4 → Pre-permission
- Pre-permission → системный запрос → Scan Loader → Paywall

- спокойствие

- технологичность  
- профессионализм  
- безопасность
