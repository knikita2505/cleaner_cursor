# [SCREEN] photos_similar

## Назначение
Экран для очистки **похожих, но не идентичных** фотографий: серийные кадры, почти одинаковые ракурсы, неудачные дубли.

---

## Структура UI

Очень похож на `photos_duplicates`, но с другим позиционированием.

### 1. Header
- Заголовок: `Similar photos`
- Back
- Кнопка `Smart select` (вместо Select All).

### 2. Summary Bar
- `Groups: <N>`
- `Suggested to delete: <K photos>`
- `Potential savings: <X.Y GB>`

### 3. Список групп
- Каждая группа — серия кадров (например, 5–20 фото с одной сцены).
- Превью: 1–3 лучшего фото + количество `+ 12 more`.

При раскрытии группы:
- Сетка фото;
- У каждого — чекбокс Delete, по умолчанию отмечены «хуже» снимки;
- Лучшее фото в группе может быть помечено бейджем `Best`.

### 4. Smart Select
- При нажатии:
  - автоматический выбор `Delete` для всех групп на основе эвристик (резкость, экспозиция, дубли из burst-съёмки).
  - если пользователь уже менял выбор в группе — не трогаем.

### 5. Bottom Bar
- Аналогично duplicates: `Delete selected`.

---

## Логика

- Эвристика похожести:
  - сравнение сцен по feature-вектору/цветовым гистограммам,
  - временная близость,
  - данные burst-серий.

- Нельзя оставлять группу полностью пустой (как минимум 1 фото должно остаться).

- Возможность «очистить только часть»:
  - пользователь может просто пройти 2–3 группы и удалить то, что выбрал.
