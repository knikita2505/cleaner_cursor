# photos_swipe_clean

## Назначение
Вовлекающая фича в стиле Tinder-свайпов: пользователь быстро пролистывает фото одно за другим, свайпая:
- вправо → удалить
- влево → оставить

Функция должна ощущаться “игровой”, но полезной.

## Основной сценарий
1. Пользователь запускает Swipe Clean из Photos overview.
2. На экране показывается по одному фото на весь экран.
3. Свайпы в стороны определяют действие.
4. В конце — summary с итогами.

## Layout

### Fullscreen Content
- Фон: тёмный.
- Photo card по центру:
  - Размер: с отступами 16–24pt от краёв.
  - Radius: 24pt.
  - Лёгкая тень.
- На карточке:
  - Сверху справа маленький label: `Screenshot`, `Burst`, `Similar` и т.п. (тип).
  - Снизу — полупрозрачный overlay с датой (`Mar 21, 2025 • 4.2 MB`).

### Top Bar
- Left: Close (X)
- Center: Progress (e.g., `12 / 120`)
- Right: Small skip icon (пропустить без действия)

### Bottom Hints
Два текста, слегка подсвеченных:
- Слева: `Keep ←` (с иконкой стрелки влево)
- Справа: `Delete →` (с иконкой корзины)

Они слегка двигаются/пульсируют пару секунд при первом открытии.

### Summary Modal (финальный)
Когда фото закончились:
- Title: `Swipe session complete`
- Subtitle: `You deleted 42 photos and kept 78.`
- CTA: `View deleted space saved`
- Secondary: `Done`

## Жесты и анимации
- Swipe right:
  - Карточка смещается вправо, слегка поворачивается.
  - Появляется красноватый overlay `Delete`.
- Swipe left:
  - Карточка смещается влево.
  - Появляется зелёный/синий overlay `Keep`.
- После отпускания пальца:
  - Если свайп достаточно большой → анимация “улетает”.
  - Если недостаточно → карточка возвращается в центр.

## Логика
- Очередь фото выбирается заранее (например, скриншоты, похожие и дубликаты).
- Все действия кешируются в session state:
  - `kept[]`
  - `deleted[]`
- В конце одной сессии — один confirmation:
  - Подтверждение удаления всех `deleted`.
- Можно ограничить длину сессии (например, 100 фото за раз).

## Edge cases
- Если фото закончились очень быстро (мало элементов):
  - Показать `You don't have many photos to clean with swipe.`

## Связанные задачи

- PHOTO-SWIPE-002 — реализация свайп-механики
- PHOTO-SWIPE-003 — массовое удаление по результатам сессии
